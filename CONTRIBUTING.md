# Contributing

欢迎提交问题、测试数据和改进建议。

## 提交代码

1. Fork 仓库并从 `main` 创建功能分支；
2. 修改后运行 `npm test`；
3. 不要提交包含设备序列号、个人路径或敏感遥控数据的捕获文件；
4. Pull Request 中说明硬件组合、串口参数、复现步骤和验证结果。

## 提交逻辑分析仪数据

- 提供符合 README 中“逻辑采集文件交换格式”的 CSV/TXT；
- 同步模式应包含 `Time`、`CH0`、`CH1`、`CH2`；
- 请注明 CH0=SBUS 输入、CH1=422+、CH2=422−，以及采样率、接地和端接方式；
- 跨隔离两侧同步测量前，必须明确说明测试阶段是否允许共地。
