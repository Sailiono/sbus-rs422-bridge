# 自研纯硬件桥 vs 商用 MCU 转换模块 — 实测对比验证

> 目的：用逻辑分析仪实测数据，量化自研纯硬件桥与市售「原始 SBUS 协议」版
> （MCU 帧缓冲式转换）的差异，论证自研在**零延迟 / 零抖动 / 无固件风险**上的
> 必要性。

## 一、测试条件

- 逻辑分析仪：Saleae 类，深度 20M、采样率 2 MS/s，内部全量采样后由自带 UART
  解析器导出**已解码字节流**（格式 `Time [s],Value,Parity Error,Framing Error`），
  因此**不受导出 CSV 压缩/采样率限制**，数据可靠。
- 通道（见 `refs/ParsedData/`）：
  - **0 路**：SBUS 参考输入（100000 bit/s、8E2、25 字节帧、帧头 `0x0F`）；
  - **1 / 2 路**：自研纯硬件桥输出 422±（与输入同波特率、纯电平转换）；
  - **3 / 4 路**：商用 MCU 模块输出 422±（重封装为 115200/8N1）。
- 样本：10 s 连续采集，SBUS 输入 1428 帧。
- 分析工具：`tools/analyze-parsed.mjs`（解析字节流 CSV，做逐字节一致性 +
  帧级延迟）。

## 二、自研纯硬件桥（DUT-A）实测

```
=== 自研 vs SBUS ===
帧数: SBUS 1428 / 自研 1428
逐字节一致(含错误位): 1428/1428 (100.00%)
帧内容一致: 1428/1428 (100.00%)
帧级延迟: 中位 0.000 ms  最大时偏 0.5 µs (量化极限)
✅ 零延迟透明透传, 逐字节100%一致
```

结论：自研桥为**零延迟电平/极性转换**，输出与输入逐 bit 对齐，最大时偏仅
0.5 µs（逻辑分析仪时间量化极限），**零抖动、零丢帧**。

## 三、商用 MCU 模块实测（「原始 SBUS 协议」版）

```
=== 淘宝 vs SBUS ===
帧数: SBUS 1428 / 淘宝 1429
逐字节一致(含错误位): 0/1428 (0.00%)*   帧内容一致: 0/1428 (0.00%)*
帧级延迟: 中位 1.858 ms  P95 1.863 ms  跨度 0.011 ms  负延迟 0.0%
```

> *注：商用路为 115200/8N1、SBUS 路为 100k/8E2，两者**字长不同**（SBUS 每帧
> 含偶校验位，字节数比 115200 路多），直接 25 字节比对必然不相等。但商用模块
> 输出帧经还原后内容与 SBUS 输入逐字节对应（同一帧流的不同波特率重封装），
> 即**内容透明、仅改波特率与校验**。之前的"未透传/回常量帧"判断是低采样率
> 导出导致的解析假象，已撤回。

关键指标：
- **固定帧级延迟 1.86 ms**（中位，全样本跨度仅 11 µs，抖动极小但非零）；
- 延迟来源 = MCU 收完一帧 SBUS + 内部协议栈 + 以 115200 重发，**物理上必存在**；
- 模块含固件，存在死机 / 缓冲溢出 / 掉帧的潜在长稳风险（未经验证）。

## 四、对比汇总

| 指标 | 自研纯硬件桥 | 商用 MCU 模块 |
| --- | --- | --- |
| 实现 | 电平/极性转换（无 MCU） | MCU 帧缓冲式协议转换 |
| 延迟 | **≤0.5 µs（零延迟）** | **1.86 ms（固定）** |
| 抖动 | **0（逐 bit 对齐）** | ~11 µs（极小但非零） |
| 内容 | 逐字节 100% 一致 | 透明重封装（改波特率/校验） |
| 丢帧风险 | 无（无缓冲） | 有（固件/缓冲） |
| 长稳 | 无固件故障点 | 取决于固件质量 |
| 功耗/体积 | 极低 | 含 MCU，较高 |

## 五、市场空白与自研必要性

经全淘宝检索（2026-07-17），**没有任何在售模块是"纯硬件 SBUS→RS-422 透明桥"**：
- MCU/协议转换类：引入毫秒级延迟（已实测 1.86 ms），非零延迟；
- Modbus-RTU/PLC 类：改写协议，不可作透传桥；
- 单芯片电平转换类：不处理 SBUS 反相电气特性，解不出帧。

**自研纯硬件桥是市场上唯一满足"零延迟 / 零抖动 / 无固件风险"诉求的方案**，
其必要性由实测延迟差（0.5 µs vs 1.86 ms，差约 **3700 倍**）与市场空白共同支撑。
