# 双 UART 模块分析工具

本工具对逻辑分析仪（如 LA2016）导出的多通道 CSV 进行通配解析，自动推断
通道角色，并对两路 UART 输出分别独立解码、对比。它是**与具体协议无关的
通用双模块分析能力**，既可用来对比两块功能相同的模块，也可用来单独分析
单块模块的多路输出。

> 适用范围说明：只有当两块被测模块**功能相同**（都是 SBUS→RS-422、不改
> 协议内容）时才有对比意义。对比方式取决于实现：
> - **零延迟透明桥 vs 零延迟透明桥**（如两块纯硬件桥）：输出与输入逐 bit
>   对齐、波特率相同，可做**逐字节 / 逐帧时域对齐**对比。
> - **零延迟透明桥 vs 帧缓冲式转换模块**（如商用「原始 SBUS 协议」版：收完
>   一帧 SBUS → 改波特率/校验为 115200/8N1 → 再发）：两者**协议内容相同**
>   但输出**晚约一帧、波特率不同**，不能逐字节时域对齐，应做**帧序列匹配
>   + 帧级延迟**对比（见下节）。
> - **协议不同的模块**（如一端透明桥、另一端把 SBUS 重封装为 Modbus-RTU
>   的模块）：输出无可比性，仅可各自独立解码，**不应**做透传对比。

## 解析能力

`src/logic-capture.js` 的 `parseLogicCaptureText`：

- 通配任意列名：列头含 `Time`/`时间` 即可，通道标签可以是 `SBUS`、
  `422+`、`淘宝422-`、`CH1`、中文或任意命名；
- 兼容 UTF-8 与 GBK/GB2312 编码（见 `docs` 中编码说明）；
- 容忍 BOM、逗号加空格分隔、空行；
- 返回 `columnLabels` 与按通道索引的 `channels`，供上层推断角色。

## 角色自动推断

`autoDetectMapping(capture)` 按标签关键字推断：

- **参考输入**：含 `sbus` / `输入` / `in` 的通道；
- **DUT-A / DUT-B 输出对**：各由一对正负通道组成，标签含
  `422+` / `422-`、`输出+` / `输出-`、`A` / `B` 等；
- 支持单组（DUT-A 自研纯硬件桥）或两组（DUT-A + DUT-B）输出。

推断结果可由 UI / CLI 参数覆盖（用户可改每路角色与信号参数）。

## 独立解码与对比

`analyzeDualModules(capture, mapping, options)` 对两路输出分别：

- 独立估算比特周期（`estimateBitPeriod`），或在用户显式指定波特率时优先使用；
- 支持可配置 `parity`（`even` / `odd` / `none`）与 `stopBits`（`1` / `2`）；
- 计算各路波特率、互补率、有效帧数、统计指标（帧率/抖动/丢帧）；
- 通过 `compareFrameSequences` 做逐帧对齐对比（仅当两路协议一致时有意义）。

## 帧缓冲式模块对比（零延迟桥 vs 帧缓冲转换模块）

当一端是零延迟透明桥、另一端是**帧缓冲式协议转换模块**（如商用「原始
SBUS 协议」版：完整收一帧 SBUS → 改波特率/校验位 → 再发）时，两者输出
**协议内容相同、但时间轴错开约一帧**，因此不能用逐字节时域对齐，而要做
**帧序列匹配 + 帧级延迟**：

1. 分别按各自波特率/校验解码两路 UART，得到各自的 SBUS 帧序列
   （每帧 25 字节，以 `0x0F` 帧头标识）；
2. 把转换模块的输出帧与参考输入帧**按内容（或顺序）匹配**——输出第 N
   帧对应输入第 N−k 帧（k 为缓冲延迟帧数）；
3. 计算**帧级延迟** = 输出帧起始时间 − 对应输入帧起始时间（应≈一帧 SBUS
   周期，约 7 ms @100k），这正体现"收完才发"的固有延迟；
4. 验证**内容一致性**：匹配帧的通道数据是否逐位相同；
5. 对比两侧的长稳（丢帧/重发）、延迟确定性（抖动）。

> ⚠ **硬性前提（采样率）**：UART 解码需每 bit 至少 ~8 采样点。SBUS=100k
> 需 ≥800 kS/s、商用 115200 需 ≥920 kS/s。早期样本（`SBUS&422+&422-.csv`、
> `2026-07-17_10-04-30.csv`）逻辑分析仪采样率仅 ~117 kS/s（最快边沿间隔
> ~8.5 µs），每 bit 仅 1 采样点——解出的"帧"虽可能以 `0x0F` 开头，但内容
> 与帧尾不可信（哪怕纯硬件透传的自研桥，其解码帧与 SBUS 输入也 0% 一致、
> 帧尾全错）。**之前"商用回常量帧、未透传"的判断是采样率不足的误判，已撤回。**
> 做帧级内容对比前须重采 ≥1.5 MS/s（建议 4~24 MS/s）且确认模块在透传模式，
> 用 `tools/verify-commercial.mjs` 复验（它会检查采样率、自动探测差分方向、
> 校验帧尾合法性与内容一致性）。

> 关键区别：自研纯硬件桥的延迟为**亚微秒（硬件门延迟）且零抖动**；帧缓冲
> 式模块的延迟为**至少一帧周期（~7 ms）且随缓冲/协议栈可能抖动**。这正是
> 纯硬件方案的低延迟、高确定性优势所在。`analyzeDualModules` 当前输出各路
> 独立指标与逐帧对比；帧级延迟匹配可由上层用两路解码后的帧时间序列计算。

## 使用方式

### Web UI

启动本地服务器后打开 `index.html`：

```bash
npm start          # python3 -m http.server 8765
# 浏览器访问 http://localhost:8765
```

1. 在「逻辑分析仪捕获（LA2016 通配格式）」上传 CSV；
2. 通道映射自动推断，可手动调整每路角色；
3. 参数区为每路选择信号参数（原 SBUS / 自定义波特率、校验、停止位）；
4. 运行分析 → 查看双模块各自指标与对比面板。

### CLI

```bash
# 单文件多通道：同一次同步采集里解码两个模块
node tools/benchmark-compare.mjs --dual <file.csv> \
  --a-baud 100000 --b-baud 100000 --a-parity even --b-parity even --a-stop 2 --b-stop 2

# 传统单 DUT / 双 DUT 模式
node tools/benchmark-compare.mjs --dut-a captures/dut-a.json --dut-a-name "DUT-A"
```

`--dual` 模式选项：

| 选项 | 默认 | 含义 |
| --- | --- | --- |
| `--a-baud` / `--b-baud` | 自动估算 | DUT-A / DUT-B 显式波特率 |
| `--a-parity` / `--b-parity` | `even` | `even` \| `odd` \| `none` |
| `--a-stop` / `--b-stop` | `2` | 停止位数 |
| `--a-name` / `--b-name` | 自动 | 模块显示名 |

报告含通道映射、各模块指标、逐帧对比（若可比）与 side-by-side 表。
报告本身就是数据摘要，**不利数据也应如实保留**。
