# 纯硬件桥与 MCU 转换模块对比测试计划

> 状态：待执行。本文记录后续验证思路，不代表已经获得测试结论。

## 为什么做这项对比

后续将引入一款商用、带 MCU 协议处理的 SBUS→RS-422 模块，与本项目不解析、不缓存且不重组协议内容的纯硬件透明桥进行同条件对比。

目标不是预先断言哪种方案更好，而是用可复现数据回答：在完成相同 SBUS→RS-422 功能时，纯硬件转换能否在数据透明性、传播延迟、时序确定性、长期稳定性和故障可解释性方面体现实际价值。

## 被测对象

- **DUT-A：本项目纯硬件透明桥**：逻辑反向、隔离和差分电平转换，不主动解析或重建 SBUS 帧；
- **DUT-B：商用 MCU 型模块**：由 MCU 接收 SBUS，再通过软件或固件处理后输出 RS-422；
- 为两个样品记录型号或样机编号、硬件版本、固件版本、供电、端接配置和工作模式。对照模块的具体品牌和型号可在实测报告中按披露需要决定。

## 公平测试原则

除被测模块外，每轮使用相同的：

- 标准 SBUS 信号源及固定、可重复的通道测试序列；
- `100000 bit/s / 8E2`、输入极性和帧率；
- 电源、线缆类型与长度、120 Ω 端接和环境条件；
- USB→RS-422 适配器、驱动、电脑、上位机版本与采集设置；
- 预热时间、单轮持续时间、重复轮数和开关机次数。

测试次序应交替或随机安排，避免温度、供电和时间漂移总是偏向同一个模块。每个工况至少重复三轮，并保留所有原始数据，包括不符合预期的结果。

## 指标与统计口径

| 类别 | 建议指标 | 说明 |
| --- | --- | --- |
| 数据透明性 | 完整帧逐字节一致率 | 输入和输出按 25 字节 SBUS 帧对齐后比较，载荷、标志位和帧尾都计入 |
| 字节错误 | 错误字节数 / 可比较总字节数 | 包含替换、缺失和多余字节；必须说明对齐算法 |
| 帧错误 | 错误帧数 / 可比较总帧数 | 分列内容不一致、丢帧、重复帧、乱序和非法帧 |
| 比特误码 | 错误位数 / 可比较总位数 | 只有输入、输出原始位流可可靠对齐时才称 BER；仅凭已解码串口字节不能覆盖起始位、校验位和停止位 |
| 稳定性 | 连续运行时间、有效帧数、同步异常、断流和重连次数 | 建议包含短测、长时间运行和多次上电循环 |
| 延迟 | 输入边沿到输出边沿的中位数、P95、P99、最小值和最大值 | 必须使用共同时基同步采集；同时报告样本数 |
| 抖动与确定性 | 延迟极差、标准差或分位区间 | 重点观察 MCU 缓存、调度或逐帧重发是否引入周期性波动 |
| 时序保持 | 输入/输出波特率、帧周期和帧间隔差异 | 检查是否存在重新定时、帧率改变或批量输出 |
| 启动行为 | 上电到首个有效输出帧的时间 | 多次上电统计中位数和最坏值 |
| 边界工况 | 供电、电缆长度、温度与干扰条件下的上述指标 | 工况范围应依据实际应用需求确定 |

## 建议的最小测试矩阵

1. **基准短测**：正常供电、短线、标准端接，验证接线、极性和统计链路；
2. **长时间稳定性**：两种模块使用相同测试序列连续运行，记录总帧数、错误、断流和重连；
3. **同步时域测试**：同时采集输入 SBUS、RS-422+ 和 RS-422−，比较传播延迟、抖动、波特率和逐帧数据；
4. **上电循环**：重复断电和上电，比较首帧时间、异常输出和恢复一致性；
5. **压力工况**：逐步改变线缆长度、供电范围或干扰条件，并一次只改变一个变量。

同步采集涉及隔离两侧时，测试接地可能暂时旁路隔离，只能用于明确允许共地的实验环境。隔离性能和同步延迟验证应分别设计，不能用共地采集结果证明电气隔离。

## LA2016 多通道同步采集与双模块解析

实测样本 `refs/接收机为FTR8B-0路SBUS输出-1路422输出+-2路422输出--3路淘宝422输出正-4路淘宝422输出负.csv`
采用单台逻辑分析仪同步采集 5 路信号，由 `src/logic-capture.js` 的 `parseLogicCaptureText`
通配解析（任意列名、中文标签、BOM、逗号或逗号加空格分隔均可），再由 `autoDetectMapping`
按标签关键字自动推断角色，最后 `analyzeDualModules` 对两路输出分别独立解码对比。

### 通道约定（样本）

| 通道 | 含义 | 角色 |
| --- | --- | --- |
| 0 | SBUS（接收机原始输出） | 参考输入 |
| 1 | 自研 422+ | DUT-A 输出正 |
| 2 | 自研 422− | DUT-A 输出负 |
| 3 | 淘宝 422+ | DUT-B 输出正 |
| 4 | 淘宝 422− | DUT-B 输出负 |

### 关键硬件差异（必须如实呈现）

- **DUT-A（自研纯硬件桥）**：只做电平/极性转换，不改变波特率，输出仍为 `100000 bit/s / 8E2`；解析得有效 SBUS 帧 71 帧，互补率与字节流与输入一致。
- **DUT-B（商用 MCU 模块）**：把 SBUS 的 100k 转换为 RS-422 的 `115200 bit/s`（实测约 113.9k），且输出并非 8E2 透传的 SBUS 字节流；按 115200/8N1 解析得到 0 个有效 SBUS 帧。这是真实特性，工具会如实记录，不强行解释。

### 工具链

- Web UI（`index.html` + `src/app.js`）：上传 LA2016 CSV → 自动推断映射 → 用户可改每路参数（原 SBUS 或自定义波特率/校验/停止位）→ 运行 `analyzeDualModules` → 显示双模块各自指标与逐帧对比；
- CLI（`tools/benchmark-compare.mjs --dual <file> --a-baud 100000 --b-baud 115200 --b-parity none --b-stop 1`）：生成 Markdown 报告，含通道映射、双模块指标与 side-by-side 对比；
- 测试（`tests/logic-capture-dual.test.js`）：构造 LA2016 风格 5 列合成数据，验证双模块解码、商用 115200/8N1 场景与逐帧对比逻辑。
