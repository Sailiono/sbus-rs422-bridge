一套面向遥控与嵌入式系统的开源 SBUS → RS-422 透明桥接方案，包含隔离硬件、浏览器上位机和可复现的验证流程。

当前仓库发布已经完成实机验证的上位机软件和第一版硬件原理图 PDF。可编辑原理图源文件、PCB、BOM 和制造文件仍在整理中；现有资料可以用于理解和评审设计，但不应视为完整的可生产硬件包。

硬件原理图：[`hardware/sbus-rs422-bridge-schematic.pdf`](hardware/sbus-rs422-bridge-schematic.pdf)

上位机通过 USB 转 422 串口读取模块输出，同时展示重建逻辑波形、SBUS 原始包、16 路通道、标志位、帧率与异常统计；还可导入符合本项目交换格式的通用三通道 CSV/TXT，分析输入/输出波特率、逐帧一致性和数字传播延迟，不依赖特定品牌的逻辑分析仪。

```text
标准反向 SBUS ──> 可选反向器 ──> 隔离 RS-422 桥 ──A/B──> USB→RS-422 ──> 上位机
       │           当前启用             │
       └── CH0 ─────── 逻辑分析仪 ────── CH1/CH2 ──┘  （仅在明确允许共地的测试阶段）
```

## 硬件功能

- **SBUS→RS-422**：把单端 SBUS 转为抗干扰能力更强的 RS-422 互补差分信号；
- **可选逻辑反向**：板上预留接收和发送两路可选反向器，可在反向 SBUS 与非反向 UART 逻辑之间转换；
- **当前配置**：外部输入使用标准反向 SBUS，原理图当前装配值启用反向器，将其转换为非反向内部逻辑后送入隔离 RS-422 收发器；
- **协议内容不变**：反向器只改变电平逻辑极性，设计目标仍是保持 `100000 bit/s / 8E2`、帧率和 25 字节数据内容不变；
- **电气隔离**：SBUS 侧与 RS-422 侧使用不同地，隔离收发器阻断两侧直接地连接；
- **模式与接口保护**：提供收发使能模式选择、差分端接和接口保护。当前项目实测与上位机验收聚焦 SBUS→RS-422 单向链路，反向发送和双向模式暂不作为已验证功能宣称。

具体器件、极性选择方式和模式表见 [`hardware/README.md`](hardware/README.md) 与[原理图 PDF](hardware/sbus-rs422-bridge-schematic.pdf)。

## 目标细化

本工具把“证明模块可用且没有改变 SBUS”拆成三个可验收层次：

1. **功能可用**：标准反向 SBUS 经当前启用的反向器和 RS-422 桥接后，电脑能以 `100000 baud / 8 data / even parity / 2 stop` 连续收到 25 字节 SBUS 帧，帧头、帧尾和 16 路通道可正确解析。
2. **数据未改变**：输入侧参考捕获与模块输出捕获经帧序列自动对齐后，每个完整 25 字节帧逐字节一致。界面支持导入参考 JSON 并给出匹配数和匹配率。
3. **电气与时序未改变**：RS-422 差分幅值、共模、上升/下降沿、终端匹配和精确位宽由示波器或逻辑分析仪验证。USB 串口只提供解码后的字节，不能替代这部分测量。

因此，只有输出侧 USB 转 422 时，本工具可以证明“模块输出是有效、可解析的 SBUS 数据”；要严谨证明“数据未改变”，还需要输入侧参考；要证明“物理波形未改变”，还需要外部仪器。

## 协议兼容性：先确认“SBUS”到底是哪一种

本项目严格验证标准的 `100000 bit/s / 8E2 / 反向逻辑 / 25 字节 / 16 通道` SBUS。设备说明中的“支持 SBUS”“兼容 SBUS 舵机”或端口标注为“SBUS”，不一定代表当前线上传输的是逐字节兼容的标准 SBUS。

| 类型 | 当前支持 | 需要注意 |
| --- | --- | --- |
| 标准 SBUS / SBUS-16 | 完整支持 | 必须同时满足波特率、8E2、极性、帧长和帧尾要求 |
| S.BUS2 | 仅识别 RC 控制帧 | 不解析遥测时隙，不能据此证明完整 S.BUS2 总线透明传输 |
| W.BUS / W.BUS2 | 不按标准 SBUS 接受 | 可能兼容舵机或部分控制数据，但扩展帧尾、附加字节会破坏严格的字节级等价 |
| SBUS-24 | 不支持 | 24 通道扩展并非标准 16 通道帧，链路全部设备都需明确支持 |
| 非反相、高速或厂商修改版 SBUS | 默认不支持 | 帧相似不等于电气、波特率和时序兼容 |
| F.Port、F.BUS、CRSF 等 | 不支持 | 属于其他串行协议；反相或使用“SBUS”丝印引脚不会把它们变成 SBUS |

项目测试确实遇到过 W.BUS 控制数据类似 SBUS、但帧尾在 `0x01–0x0F` 间循环并带附加字节的情况。这类信号可能足以驱动兼容舵机，却不能用于证明标准 SBUS 的逐字节透明传输。完整的兼容性矩阵、S.BUS2 支持边界和排查顺序见 [`docs/protocol-compatibility.md`](docs/protocol-compatibility.md)。

## 硬件连接与采集

```text
SBUS 信号源 ──> 被测 SBUS→422 模块 ──A/B──> USB→422 ──> 电脑
                                           GND（按适配器说明连接）
```

1. 确认 422 A/B 定义。不同厂商对 A/B、`+/-` 命名可能相反；无数据时先核对说明书，必要时交换 A/B。
2. 确认总线端接。长线或高速边沿测试时按设计要求使用约 120 Ω 终端，不要在多个位置重复端接。
3. 启动页面，点击“连接串口”，选择 USB 转 422 设备。页面固定使用 SBUS 的 `100000 / 8E2`。
4. 至少连续采集数百帧，观察有效帧、帧率、同步异常、`FRAME LOST` 和 `FAILSAFE`。
5. 导出 JSON 原始捕获、CSV 通道数据和 HTML 验证报告。

### 软件验证状态

当前版本已经完成标准 SBUS 连续采集、25 字节帧解析、通道与标志位解码、输入/输出逐字节对比，以及同步和隔离分次逻辑采集分析。公开仓库不绑定具体测试设备；不同硬件组合应按照下文统一格式提交捕获文件和验收记录。

界面把“初始对齐字节”和“运行期同步异常”分开统计，并用连续两帧确认首次锁定，避免载荷中的 `0x0F` 被误认成帧头。串口连接可能从任意帧内字节开始，首次找到完整 `0x0F + 25 bytes + 合法帧尾` 之前跳过若干字节属于正常锁定过程；首次锁定后出现的丢弃字节才计入运行期异常。

浏览器实时采集使用 64 KiB Web Serial 接收缓冲。界面中的同步异常覆盖模块、RS-422/USB 适配器、驱动和浏览器接收的整条链路，单凭该计数不能直接定位故障来源。

## “数据未改变”的建议验收方法

### 方法 A：固定测试序列（现有硬件最容易落地）

1. 让 SBUS 信号源重复输出一个已知、确定的通道序列。
2. 用支持 SBUS 反相和 `100000 8E2` 的输入侧适配器采集源数据，并“导出捕获”。
3. 接入被测模块与 USB 转 422，采集模块输出。
4. 在“逐字节一致性”处导入输入侧 JSON。工具会在前后 100 帧内寻找最佳偏移，然后比较所有可对齐的 25 字节帧。
5. 匹配率为 100% 才能对本次捕获给出“逐字节未改变”的结论。

## 通用逻辑采集文件分析

上位机只约定数据交换格式，不要求逻辑分析仪品牌或配套软件。支持扩展名为 `.csv` 或 `.txt` 的文本文件，编码建议使用 UTF-8。

### 逻辑采集文件交换格式

- 每行表示一个采样点或逻辑状态变化事件；
- 第一列为时间，表头使用 `Time` 或 `时间`，可在名称中注明 `s`、`ms`、`us`/`µs` 或 `ns`；未注明时按秒处理；
- 通道列使用 `CH0`、`CH1`、`CH2`，也接受 `Channel 0`、`通道 0` 等等价名称；
- 逻辑值可使用 `0/1`、`L/H`、`Low/High` 或 `False/True`；
- 分隔符支持逗号、制表符、分号或连续空白；
- `CH0` 为反向逻辑的 SBUS 输入，`CH1` 为 RS-422 正端，`CH2` 为 RS-422 负端；
- 时间可升序或降序，上位机会自动归一化为相对时间。

最小示例：

```csv
Time(s),CH0,CH1,CH2
0.000000,0,1,0
0.000010,1,0,1
0.000020,0,1,0
```

### 同步单 CSV（测试阶段推荐）

测试阶段如果已经明确允许 SBUS 输入地与 422 输出地共地，可用同一台逻辑分析仪同步采集：CH0 接 SBUS 输入，CH1 接 422+，CH2 接 422−。导出同时包含三个通道且符合上述交换格式的原始数字采样文件，然后点击上位机的“导入同步 CSV”。

同步模式保留三个通道的共同时间轴，可直接给出：

- CH1/CH2 差分互补率和输出正常极性；
- 输入、输出独立估算波特率及差异；
- 输入与输出完整 25 字节 SBUS 帧逐字节一致率；
- 输入边沿到输出边沿的传播延迟中位数、P95、最小值、最大值和样本数；
- 三通道同时间轴数字波形。

### 隔离双文件

如果测试目标是验证模块隔离，不能让共地的逻辑分析仪同时跨接输入和输出两侧，否则测量系统会旁路隔离。此时使用两次独立采集：

1. 让 SBUS 信号源循环发送固定、可重复的测试序列。
2. 输入侧采集：断开逻辑分析仪的所有输出侧连线；仪器 GND 只接 SBUS 输入地，CH0 接 SBUS 信号，导出 CH0 CSV/TXT。
3. 输出侧采集：完全移除输入侧 GND 和 CH0；仪器 GND 改接隔离后的 422 地，CH1 接 422+，CH2 接 422−，导出 CH1/CH2 CSV/TXT。
4. 在上位机的“隔离双文件”区域依次导入两侧文件，工具按完整 SBUS 帧内容自动对齐。

分次采集能够证明数据内容和两侧波特率一致，但不能测传播延迟。要在保持隔离时测量传播延迟，需要两套彼此隔离且共用时间基准的采集设备，或额定隔离的差分/隔离探头。如果 USB 转 422 的 GND 已接模块输出地，进行输入侧采集前还应断开该 GND/USB 连接或使用合规 USB 隔离器。

## 波形含义

界面同时画两条逻辑线：

- `USB-UART 逻辑`：USB 转 422 已解码字节所对应的普通 UART 逻辑；
- `原始 SBUS 逻辑`：按 SBUS 反向电平规则对上一条逻辑取反得到的推导结果。

每字节按 `1 start + 8 data (LSB first) + even parity + 2 stop` 重建，每位固定显示为 10 μs。它适合解释帧内容和位序，不是示波器实测。帧率和抖动采用 USB 数据到达时间；适配器批量上报时，界面会标注“间隔含估算”。
